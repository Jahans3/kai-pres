Fundamentals: Under the Hood: What is programming?
Programming is simply writing out a series of instructions for a computer to carry out
A programming language is a style of instruction targeted towards a specific runtime environment, eg. Java or Scala for the JVM
Programming constructs are largely the same across all languages, with usually only syntactic differences
Fundamentals: Under the Hood: From human-readable to machine code
It's easy to forget but the computer cannot understand the code you write
For a computer to be able to understand what you are telling it to do, the code must go through a build process that transforms the code into machine code
You won't need to know anything beyond the basics unless you go into advanced programming, but it is very useful to understand the process
Compiled languages:
  • 1. You write out some code in a text file
  • 2. The text files are read using a compiler specific to the language you used
  • 3. The compiler runs a parser that runs once over your code without executing any commands
  • 4. The source code (the code you wrote) is transformed into machine code
  • 5. The machine code can be executed by the computer to run the application
Interpreted languages:
  • 1. You write out some code in a text file
  • 2. At the time of execution (not prior, like compiled languages) the code is read by a JIT (Just In Time) compiler, called an interpreter
  • 3. The interpreter first runs a parser, the parser runs once, and simply looks over the code without executing any commands
  • 4. The code will be transformed into something the engine running it can understand, usually bytecode
Programming languages follow standards set out by official specifications, there may be different compilers for the same language, but as long as they follow the spec correctly it will work

Eg. JavaScript runs on V8 in Chrome and Chakra in Edge
Fundamentals: Terminating Lines
In the vast majority of languages you must terminate each line of code with a semi-colon, this is so the compiler knows where each line starts and ends.
This is not true for JavaScript, but it is strongly recommended that you do use semi-colons, as there are some (albeit rare) cases where this can causes bugs.
Fundamentals: Operators
Operators are largely universal, with only slight differences between programming languages.

You probably know most of these already...
Multiply: *

5 * 5 = 25
Subtract: -

10 - 5 = 5;
Add: +

10 + 5 = 15;
1 + 1 = 3;
Divide: /

10 / 5 = 2;
Modulus: %

Take the first number, divide by the second, and give the remainder

7 % 3 = 1
Equal to: ==

5 == 5 = true;
Equal to and of the same type: ===

5 === '5' = false;

Note that the second '5' here is in quotes, meaning it is a String, there it is a different type to 5, which is a Number
Is not equal to: !=

5 != '5' = false;

Note that here we are not checking the type of the second 5, so it is inferred (coerced) into a Number type to check against the 5
Is not equal to and not the same type: !==

5 !== '5' = true;

Here we do check the type, so "5 does not equal five AND is not the same type" results in true, because although both are 5, one is a type of String, while the other is a Number
Greater than: >

5 > 6 = false;
Lesser than: <

5 < 6 = true;
Greater or equal to: >=

5 >= 5 = true;
Lesser or equal to: <=

5 <= 6 = true;
Fundamentals: Variables
Variables are programming constructs that hold data

There are different types of variables with different behaviours, but they all act as data stores
Think algebra:

Where the variables A = 5 and B = 2, work out A * B
To create a variable you must declare it, the following JavaScript example stores the word "Hello" to a variable called greeting

let greeting = 'Hello';
The value of greeting can be accessed at any time:

console.log( ); - This tells the computer to print (log) the value of a given piece of data to the console (the matrixy thing/command line)

console.log(greeting); - Result: "Hello"
The value of greeting can also be changed:

greeting = "Goodbye";

console.log(greeting); - Result: "Goodbye"

Note the lack of "let" because the variable has already been defined, and we are just changing the value
Fundamentals: Data Types
When dealing with data we must distinguish between data types, this is essential to programming
A data type is a classification for the type of data which a variable will hold

There are many different data types
Typed Languages
  • Author must explicitly declare the type of each variable
  • Compiler will check no types are mixed or changed unintentionally
  • To change the type of a variable the author must cast from one type to another
  • Adds extra layer of safety - developer will catch errors before runtime
Interpreted Languages
  • Author does not explicitly set the data type, instead the interpreter infers it from the code
  • More error-prone than type-safe languages
  • Types can be changed on the fly
  • eg. JavaScript
Data types take up different amount of data (typically):
  • Int: 2-4 bytes
  • Float: 4 bytes
  • Double: 8 bytes
  • Long: 10 bytes
Types:

Types will vary slightly by language, but typically you will have:
  • Integer - A whole number
  • Float - A number where the decimal can be anywhere in the number
  • Double - A Float with twice the maximum value
  • Boolean - One of two values: true or false
  • String - A collection of characters, text
  • Long, Char, Byte, and others for more advanced memory management and numerical reasons
Fundamentals: Constructs
A programming or data construct is a piece of data constructed of other, smaller types of data
The first construct you will probably see will be an array, defined using square brackets []
An array is an object that simply stores multiple pieces of a data in a sequence, with each data piece separated by a comma, here is one being assigned to the variable myArray:

let myArray = [5, 10, 15];
Each piece of data in an array is called an element and is accessed by an index

let myArray = [5, 10, 15];

console.log(myArray[0]); - Result: 5

Important: In computing terms, we begin counting at 0, so to access the first element of myArray we used 0.
Another common construct is the Object, this is a tree of key:value pairs and is denoted using curly braces. Here is one being assigned to the variable myObject:

let myObject = {
propertyOne: 'value1',
propertyTwo: 500,
};

Think of each property as a variable, meaning you can assign any value you like to it.
To access the value of a property of an object use dot notation:

console.log(myObject.propertyOne); - Result: "value1"
Another common construct is a function, more on these in a bit
Conditionals
A conditional is an evaluation based on the result of a given condition
The most simple conditional is an if statement


If statements are blocks of code that are evaluated "if" the given condition is true
if (5 == 5) { console.log('Wow, 5 really does equal 5!'); };

Result: "Wow, 5 really does equal 5!"
Iterables and Iterable Constructs
As we saw, constructs contain multiple values, to access these values programmatically we must iterate over the construct
For loops

A for loop executes the code contained within "for" the given amount of times

for (let i = 0; i < 5; i++) { console.log(i); } - Result: 0, 1, 2, 3, 4
We can use the value in our loop to access values by their index:

let myArray = ['fuck', 'a', 'duck'];

for (let i = 0; i < myArray.length; i++) {
console.log(myArray[i]);
}

Result: "fuck" "a" "duck"
Loops can take many forms, but they all follow the same fundamental principle of executing a block of code over and over
Other types of iterables:
  • while loop - keeps looping "while" a given condition is true
  • Iterable methods - functions belonging to specific objects designed to iterate over objects (usually of the type they are attached to)
  • Recursive functions - recursion in maths is the repeated application of a procedure - exactly the same in computer science. This is a function that calls itself.
Functions
Functions are essential to programming, they are blocks of code that can be executed on-demand with given parameters that will modify the result.

It is the programming equivalent of "function machines" from school, where you give some numbers to the machine and it gives you something different back based on the given values, except where function machines have access to theoretical numbers, computational functions have access to the system they are running on
Parameters are the arguments given to a function to be used by the function
Take this function, myFunction, that accepts the argument "number" and adds 1, then prints it:

function myFunction(number) {
number = number + 1;
console.log(number);
};

Now if we call the function and pass in 7 as a result, what happens?
Looking at our functions in mathematical terms can make it easier to understand...
Where the JavaScript function myFunction:

function myFunction(number) {
number = number + 1;
console.log(number);
};


Is represented in mathematical notation (f of X) as:

f(X) = X + 1
Where X is 7, f of X evaluates to 8 in JavaScript just as it does in maths:

f(X) = 7 + 1 = 8
myFunction(X); - Result: 8
Programming for the web: JavaScript
JavaScript is:

  • Interpreted
  • Loosely-typed, ie. you don't specify data types, the interpreter infers them
  • Runs in the browser
  • Has deep integration with browsers, allowing you do to do almost anything you want
  • Follows the ECMAScript spec
  • String and independent: don't need no man
JavaScript does not require you to specify data types when defining variables, but there are different types of variable

const - declare a variable and do NOT allow the value to be changed, the value is "constant"
let - declare a variable that can be changed
var - has very specific use-cases, can forget about this one for now
Since JavaScript is loosely-typed, but you still need to be able to check types, they are broken down into simple categories:
  • Boolean - true/false
  • String - text
  • Number - numbers. In JavaScript this value is a number between +2^53 - 1 and -2^53 - 1

These categories are known as "primitives"
  • let myBoolean = true;
  • let myString = 'Hello!';
  • let myNumber = 42;
Functions in JavaScript are treated as first-class objects, meaning they can be stored in variables, passed as parameters to other functions, and are able to be treated like normal data types.

This is not true for a lot of languages, meaning you have more freedom when writing JS.
Functions:

Declared using the "function" keyword, or the lambda symbol, "=>"
Different function declarations have different implications, for now we will focus on "function"
Functions:
Declare a function by writing the keyword "function" followed by the function name, then brackets, if you want to pass parameters, you should define them in the brackets:

function sayHello(name){
console.log(`Hello, ${name}!`);
};
Functions:
Execute (call) a function by writing out the name (known as a symbol) and putting brackets next to it, placing any parameters between the brackets:

sayHello('kai');
Result: "Hello, kai!"
Functions:
Functions can be stored inside variables:

let sayHello = function (name){
console.log(`Hello, ${name}!`);
};
